# 15 — 독후감 제출 + 동호회 피드 + 미제출 관리

> 작성일: 2026-04-24
> 상태: v1 구현 완료

---

## 1. 왜 만들었나 (Problem)

- 무제는 매달 구매한 책마다 **독후감** 을 제출하는 것이 참여 규칙이다.
- 구글시트 운영 시절에는 링크를 시트에 붙이는 방식 → 제출 여부 확인이 수동, 놓치는 사람 추적 어려움.
- 더불어 "다른 사람들은 어떤 책을 읽었나" 를 보고 싶어하는 니즈가 있어 **피드** 를 함께 갖춘다.

## 2. 목표 (Goal)

- 수령(RECEIVED) 된 책에 대해 본인이 독후감을 남기고 수정할 수 있다.
- 클럽 멤버 누구나 피드를 본다 (비공개 옵션은 v2).
- 관리자는 **미제출자 리스트** 를 조회해 리마인더를 보낼 수 있다.

## 3. 정책 / 규칙 (Rules)

- **제출 대상**: `BookRequest.status = RECEIVED` 인 건만.
- **제출 기한**: 수령 후 정해진 일수(초기: 월말). 기한 이후는 늦은 제출로 플래그 — 벌점 정책은 v2.
- **가시성**: 같은 클럽 멤버 전체. 본인 외에는 수정 불가.
- **별점 제거**: 초기 설계에 있었으나 "서로 점수 주는 건 부담" 피드백으로 **본문 텍스트만** (04/기존 청크 7+8 커밋 메시지 참고).
- **링크 허용**: 외부 블로그 URL 만 첨부한 간이 제출도 인정 (v1).

## 4. UX 흐름

### 4-1. 회원 — 독후감 페이지 (`BookReportPage`)
1. 탭 1 "내 독후감" — 내가 수령한 책 리스트 + 제출 상태.
2. 미제출 책에 [작성] 버튼 → 본문/링크 입력 → 제출.
3. 제출한 건 클릭 → 피드의 해당 카드로 포커스.
4. 탭 2 "전체 피드" — 최근순 카드. 제목/저자/저자+글쓴이 표시.

### 4-2. 관리자 — 미제출자 패널 (`AdminBookReportsPage`)
- 월별 필터. 미제출 회원 표 + [리마인더 발송] 버튼 (NotificationService 경유, 12 문서).

## 5. API 표면

### 회원
- `GET  /api/v1/clubs/{clubId}/book-reports/my` — 내 제출 현황.
- `GET  /api/v1/clubs/{clubId}/book-reports` — 클럽 피드.
- `POST /api/v1/clubs/{clubId}/book-reports` — 생성.
- `PATCH /api/v1/clubs/{clubId}/book-reports/{id}` — 수정.

### 관리자
- `GET /api/v1/admin/clubs/{clubId}/book-reports/missing` — 미제출자.

## 6. 수용 기준 (Acceptance)

- [x] RECEIVED 이 아닌 책에는 독후감 생성 불가 (400).
- [x] 타인 독후감 수정 시 403.
- [x] 피드가 같은 클럽 멤버에게만 보임.
- [x] 관리자 missing 리스트가 (월, 회원) 단위로 정확.
- [ ] 제출 기한 초과 자동 플래그 + 벌점 연계 (v2).
- [x] 본문 또는 링크 중 하나만 있어도 제출 성립.

## 7. 오픈 이슈 / 향후 계획

- **댓글/좋아요**: 피드 상호작용. 운영 리스크(부담/싸움) 감안해 기본 off.
- **AI 요약**: 길이 늘어나면 카드에서 본문 첫 문단만 보여주고 전문은 상세 페이지.
- **통계**: 회원별 연간 독서 권수/글자수 요약.
- **벌점 자동화**: 미제출자 리마인더 후 일정 기간 지나면 벌점 자동 부여.
