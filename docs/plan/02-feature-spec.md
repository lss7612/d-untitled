# 기능 정의서 (TO-BE)

> 구글 시트 기반 수동 운영을 전면 대체하여 관리자 부담을 최소화하고 회원 경험을 개선한다.

## Phase 1 — 인증 & 회원 관리

### F-00: 인증

#### 기능 개요
Google OAuth 2.0 기반 로그인과 이메일 Verify Code 2-step 인증을 통해 허용된 회사 이메일 소유자만 서비스에 접근할 수 있도록 한다.

#### 비즈니스 규칙

- 허용 도메인: `kr.doubledown.com`, `afewgoodsoft.com`, `doubleugames.com` — 외 도메인 즉시 차단
- Verify Code: 6자리 숫자, 유효 시간 5분
- 코드 오입력 5회 이상 → 해당 이메일 5분 잠금 (확정)
- Google OAuth 성공 후 이메일 Verify Code 미완료 상태면 접근 제한 (2-step 미완료 상태 유지)
- 이메일 Verify Code 인증은 **최초 1회만 수행**한다. 인증 완료 후 `Member.emailVerified = true`로 영구 유지하며, 이후 재로그인 시 건너뛴다. 목적은 회사 이메일 도메인의 실제 소유 여부 확인이다 (확정)
- JWT 만료 정책: **Sliding 1시간** — 인증된 API 호출 시마다 만료 시간이 갱신되며, 비활동 1시간이면 만료된다. 만료 시 재로그인 필요. Refresh Token 미도입 (확정)

#### 권한 구분

| 기능 | 일반 회원 | 관리자 |
|------|-----------|--------|
| 로그인 | O | O |
| 이메일 인증 | O | O |
| 타 회원 계정 잠금 | X | O |
| 회원 권한 변경 | X | O |

#### Acceptance Criteria

- [ ] 이미 이메일 인증 완료된 사용자가 재로그인(Google OAuth) → 이메일 인증 건너뛰고 /home으로 이동
- [ ] JWT 만료(1시간) 후 보호 페이지 접근 → /login으로 리다이렉트
- [ ] 토큰 없이 보호 라우트(/home, /clubs, /muje) 접근 → /login으로 리다이렉트
- [ ] 토큰 있지만 이메일 미인증 상태로 보호 라우트 접근 → /auth/email-verify로 리다이렉트
- [ ] 이메일 인증 코드 오입력 5회 → 5분 잠금 (F-00 기준)

### F-00-B: 회원 프로필 관리

#### 기능 개요
회원이 본인 프로필(닉네임, 부서, 슬랙 사용자 ID)을 확인하고 수정한다. 관리자는 전체 회원 목록 조회 및 권한 변경이 가능하다.

#### 비즈니스 규칙

- 이름은 Google 계정 이름 자동 동기화 — 직접 수정 불가
- 슬랙 사용자 ID 미등록 시 개인 DM 알림 발송 불가 (채널 공지만 수신)
- 관리자는 `ADMIN` / 일반 회원은 `MEMBER` — 역할은 관리자만 변경 가능
- **최초 ADMIN 임명**: 별도 UI 없이 운영자가 DB 시드(마이그레이션 또는 직접 SQL)로 등록한다. 이후 신규 ADMIN 임명은 기존 ADMIN의 권한 변경 API로 진행 (확정)

#### 권한 구분

| 기능 | 일반 회원 | 관리자 |
|------|-----------|--------|
| 본인 프로필 조회/수정 | O | O |
| 전체 회원 목록 조회 | X | O |
| 회원 역할 변경 | X | O |

## Phase 2 — 무제 핵심 기능 (Google Sheets 완전 대체)

### F-01: 책 신청 폼

#### 기능 개요
회원이 알라딘 URL을 입력하면 도서 정보가 자동 파싱된다. 당월 예산 범위 내에서 1권 이상 신청 가능하며, 관리자가 신청 마감 처리를 하기 전까지 수정/취소가 가능하다.

#### 비즈니스 규칙

- URL 파싱 대상: 알라딘(`aladin.co.kr`) 전용 — 타 도메인 입력 시 오류 안내
- 동일 월 내 동일 ISBN 중복 신청 불가
- 잔여 예산 초과 신청 불가 — 신청 시점에 잔여 예산 실시간 표시
- 관리자가 신청 마감 처리 후 수정/취소 불가
- 외서 환율 처리: **신청 시점 환율 1개를 임시 적용** (확정). 외부 환율 API 또는 일일 고정 환율을 조회해 신청 레코드의 `exchangeRate` 필드에 동결한다. 동일 도서를 다음 달 재신청해도 환율은 신청 시점 기준으로 다시 잡힌다. Phase 2 후반에 환율 캐시 정책을 정교화한다
- 카테고리: 문학, 인문, 자기계발, 예술, IT, 만화, 경제, 라이프스타일, 과학, 기타

**예산 규칙**
- **가입월 (신입 첫 달, `Member.createdAt`의 YearMonth)**: 30,000원 한도
- **그 다음 월부터**: 35,000원 한도 (자동 전환). 가입 후 첫 신청까지 N달이 지났더라도 가입월이 아니면 35,000원 적용 (확정)
- 가입월에 책을 신청하지 않으면 한도는 자동 소멸. 다음 달 잔여로 이월하지 않는다
- 동일 월 복수 권 신청 가능 — 합산 금액 기준 예산 초과 시 마지막 신청 차단
- 신청 페이지에 잔여 예산 실시간 표시 (예: 잔여 12,000원)

#### 권한 구분

| 기능 | 일반 회원 | 관리자 |
|------|-----------|--------|
| 전체 신청 목록 조회 | O | O |
| 본인 신청 등록/수정/취소 | O (마감 전) | O |
| 타 회원 신청 수정/삭제 | X | O |
| 신청 마감 잠금 처리 | X | O |

### F-02: 주문 자동화

#### 비즈니스 규칙

### F-05: 촬영 일정 관리

#### 기능 개요
관리자가 월간 단체 사진 촬영 일정을 등록/수정/삭제한다. 일정 변경 시 전체 회원에게 슬랙 공지가 자동 발송된다.

#### 비즈니스 규칙

- 일정 수정 시 기존 등록자 전원에게 슬랙 알림 자동 발송
- 동일 날짜 동일 공간 중복 등록 불가
- 촬영 불참 벌점 부여: 관리자가 F-07을 통해 수동 부여

### F-07: 벌점 관리

#### 기능 개요
관리자가 미제출·불참·미수령 등 사유별로 벌점을 수동 부여한다. 벌점은 동호회별로 독립 운영되며, 포인트(F-11)와 별도로 집계된다. 회원은 벌점 이력을 조회하고 이의를 제기할 수 있다.

#### 비즈니스 규칙

- 벌점은 동호회별로 독립 운영된다. 타 동호회 벌점과 합산하지 않는다.
- 벌점과 포인트는 별도로 집계한다. 포인트 총점에서 벌점을 자동 차감하지 않는다.
- [미결] 포인트로 벌점 삭감 가능 여부는 추후 결정한다.
- 벌점 부여 항목: 독후감 미제출, 촬영 불참, 도서 미수령 (각 -1점)
- [결정 필요: 항목별 점수 가중치 차등 적용 여부]
- 동일 회원 동일 월 동일 항목 중복 부여 불가
- 이의 제기: 처리 중 → 수락(벌점 취소) / 기각(벌점 유지)
- 이의 제기 처리 시 해당 회원에게 슬랙 DM 발송

#### 권한 구분

| 기능 | 일반 회원 | 관리자 |

| 본인 벌점 이력 조회 | O | O |
| 전체 벌점 이력 조회 | X | O |
| 벌점 부여/취소 | X | O |
| 이의 제기 | O | X |
| 이의 제기 처리 | X | O |

### F-11: 포인트 / 벌점 시스템

#### 기능 개요
회원의 활동 기여도를 포인트로 수치화한다. 포인트는 동호회별로 독립 운영되며, 동호회 내 참여 지표로만 기능한다. 월간 랭킹을 통해 참여 동기를 부여한다.

#### 비즈니스 규칙

- 포인트는 동호회별로 독립 운영된다. 타 동호회 포인트와 합산하지 않는다.
- 포인트와 벌점은 별도로 집계한다. 포인트 총점에서 벌점을 자동 차감하지 않는다.
- [미결] 포인트로 벌점 삭감 가능 여부는 추후 결정한다.
- 포인트 기본 부여 기준: 독후감 제출 +1, 조기 제출(마감 3일 전) +1, 촬영 참여 +1
- 포인트 환전/교환 기능: 현재 로드맵에서 제외. 포인트는 동호회 내 참여 지표로만 운영한다.
- 랭킹 기준: 해당 동호회 내 당월 누적 포인트 합산
