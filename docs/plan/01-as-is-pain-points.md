# 현황분석 & 페인포인트

## AS-IS 현황 분석

- 분석 기간: 2022년 ~ 2025년 (#무제_2022 슬랙 채널 기반)
- 회원 규모: 최대 45명 (평균 활동 인원 20~30명)

### 월간 운영 사이클

1. 촬영 일정 공지 (운영진)
2. 책 신청 접수 (구글 시트 수기 입력)
3. 주문 취합 및 주문 처리
4. 인포 수령 (도서 수령)
5. 독후감 마감 및 제출 확인
6. 벌점 부여 (독후감 미제출 / 촬영 불참 / 인포 미수령)
7. 월간 운영 보고서 작성

### 현재 사용 도구

- **슬랙** (#무제_2022): 공지, 리마인더, 책 신청 안내, 벌점 고지
- **구글 시트**: 책 신청 관리, 주문 취합, 인포 수령 확인, 벌점 이력 기록
- **카카오톡**: 비공식 커뮤니케이션 및 긴급 공지

## 핵심 페인포인트

### [P1] 책 신청 관리 — 가장 빈번한 오류 발생 지점

- 구글 시트 수기 입력 → 오탈자, 카테고리 누락, 중복 신청 빈번
- 외서 환율을 신청서 · 주문서 · 회계 3곳에 각각 수동으로 기록
- 복수 신청 시 합산 주문 로직이 복잡해 수동 계산 오류 발생
- 재고 · 품절 여부 사전 확인 불가 → 주문 후 반품 처리 발생
- 월 100권 이상 처리 → 운영진 수작업 부담 극심

### [P1] 촬영 일정 관리 — 조율 비용 과다

- 13층 / 16층 촬영 공간마다 담당 admin이 달라 각각 별도 조율 필요
- 일정 변경 시 관련 인원 전원에게 수동으로 재공지
- 담당자 지정 및 @mention 수동 처리 → 잘못된 일정 공지 사고 발생

### [P1] 독후감 관리 — 리마인더 전량 수동

- 마감 D-7, D-3, D-1 리마인더를 운영진이 직접 @mention 발송
- 제출 여부를 매번 시트에서 수동 확인 → 누락 발생
- 미제출자 벌점 부여도 수동 처리 → 귀책 논란 발생 가능성

### [P2] 인포 수령 관리 — 패널티 기준 불명확

- 도서 도착 당일 수령 원칙이나 별도 알림 시스템 없음
- 1주일 방치 시 벌점 부여 규정이 있으나 자동 감지 불가 → 수동 체크
- 미수령자 파악에 시간 소요 → 운영진 공지 지연 발생

### [P2] 벌점 관리 — 이력 추적 어려움

- 독후감 미제출 · 촬영 불참 · 인포 미수령 3가지 항목을 운영진이 수동 부여
- 누적 벌점 이력이 구글 시트에 분산 → 장기 추적 어려움
- 부여 기준이 암묵적 → 이의 제기 시 귀책 논란

### [P3] 도서 대여 관리 — 이력 소실 위험

- 도서 대여 이력을 구글 시트로 수동 관리
- 도서 장기 방치 시 자동 알림 없음 → 운영진이 직접 촉구
- 비회원이 대여하는 사례 발생 → 공통 기준 마련 필요
