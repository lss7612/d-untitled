# 페이지 정의서

> 로그인부터 동호회 입장까지의 핵심 화면 정의.
> 각 페이지의 목적 / 구성 요소 / 사용자 흐름 / 접근 권한을 기술.

---

## PAGE-01: 로그인 페이지

- **목적**: 서비스 진입점. Google OAuth 2.0 인증
- **접근 권한**: 비인증 사용자만 접근 (인증 완료 시 동호회 목록 페이지로 리다이렉트)

### 구성 요소

- 서비스 로고 / 타이틀
- "Google 계정으로 로그인" 버튼 (Google OAuth 2.0)
- 회사 이메일 도메인 제한 안내 문구 (kr.doubledown.com / afewgoodsoft.com / doubleugames.com)

### 사용자 흐름

1. 로그인 버튼 클릭 → Google OAuth 팝업
2. 인증 성공 + 허용 도메인 → 이메일 Verify Code 입력 화면으로 이동 (2-step)
3. 인증 성공 + 미허용 도메인 → 오류 메시지 ("등록된 회사 이메일만 이용 가능합니다")
4. 인증 실패 → 오류 메시지 및 로그인 페이지 유지

### Acceptance Criteria

- [ ] "Google 계정으로 로그인" 버튼 클릭 → Google OAuth 인증 화면으로 이동
- [ ] 허용 도메인(kr.doubledown.com / afewgoodsoft.com / doubleugames.com) 계정으로 인증 성공 → /auth/email-verify 이동
- [ ] 미허용 도메인 계정으로 인증 시도 → "등록된 회사 이메일만 이용 가능합니다" 에러 메시지 + 로그인 페이지 유지
- [ ] Google OAuth 취소 또는 실패 → 에러 메시지 표시 + 로그인 페이지 유지
- [ ] 이미 로그인 + 이메일 인증 완료된 사용자가 /login 접근 → /home으로 자동 리다이렉트

## PAGE-01-B: 이메일 Verify Code 입력 페이지 (2-step)

- **목적**: Google OAuth 완료 후 2차 인증. 회사 이메일 소유 여부 최종 확인
- **접근 권한**: Google OAuth 완료 + 이메일 미인증 상태 사용자만 접근

### 구성 요소

- 안내 문구: "회사 이메일로 발송된 인증 코드를 입력하세요"
- 인증 코드 입력 필드 (6자리 숫자)
- 코드 재발송 버튼 (60초 쿨타임)
- 제출 버튼

### 사용자 흐름

1. Google OAuth 완료 직후 자동 진입
2. 회사 이메일로 6자리 코드 자동 발송 (페이지 진입 시)
3. 코드 입력 → 제출 → 성공 시 동호회 목록 페이지로 이동
4. 코드 불일치 → 오류 메시지 ("인증 코드가 올바르지 않습니다. 남은 시도: N회")
5. 5회 오입력 → 5분 잠금 안내 메시지 노출

### 예외 처리

- 코드 유효 시간(5분) 초과 → "인증 코드가 만료되었습니다. 재발송 버튼을 누르세요"
- 재발송 쿨타임(60초) 중 재발송 시도 → 버튼 비활성화 + 남은 시간 표시

### Acceptance Criteria

- [ ] 페이지 진입 시 인증 코드 자동 발송 (중복 발송 없이 1회만)
- [ ] 6자리 숫자 코드 정상 입력 후 제출 → 인증 성공 → /home 이동
- [ ] 코드 불일치 → "인증 코드가 올바르지 않습니다. 남은 시도: N회" 메시지 표시
- [ ] 5회 오입력 → "인증 시도 5회 실패로 잠금되었습니다. 5분 후 다시 시도해주세요" 메시지 + 입력/제출/재발송 버튼 비활성화
- [ ] 잠금 상태에서 제출 시도 → 즉시 거부 + 잠금 안내 메시지
- [ ] 코드 5분 만료 후 제출 → "인증 코드가 만료되었습니다. 재발송 버튼을 누르세요" 메시지
- [ ] 재발송 버튼 클릭 → 새 코드 발송 + 60초 쿨타임 시작
- [ ] 쿨타임 중 재발송 시도 → 버튼 비활성화 + "코드 재발송 (N초)" 표시
- [ ] 6자리 미만 입력 상태에서 제출 버튼 비활성화
- [ ] 이미 이메일 인증 완료된 사용자가 접근 → /home으로 자동 리다이렉트

## PAGE-02: 동호회 목록 페이지

- **목적**: 로그인 후 첫 진입 화면. 가입된 동호회 목록 표시
- **접근 권한**: 로그인 필수. 미인증 시 로그인 페이지로 리다이렉트

### 구성 요소

- 상단 헤더: 서비스명 / 내 정보 버튼 / 로그아웃 버튼
- 동호회 카드 목록 (현재: 무제 1개) — 카드 내 정보: 동호회명 · 한줄 소개 · 진입 버튼

### 사용자 흐름

1. 로그인 완료 → 이 페이지 자동 이동
2. 무제 카드 클릭 → 무제 Welcome 페이지로 이동
3. 내 정보 버튼 클릭 → 내 정보 페이지로 이동

---

## PAGE-03: 무제 Welcome 페이지 (대시보드)

- **목적**: 무제 동호회 대시보드. 주요 기능 진입점 역할
- **접근 권한**: 무제 동호회 회원만 접근. 비회원 접근 시 동호회 목록으로 리다이렉트

### 구성 요소

- 인사 문구: "안녕하세요, {이름}님" + 이번 달 잔여 예산 표시
- 이번 달 주요 일정 요약 (책 신청 마감, 독후감 마감, 촬영일)
- 주요 메뉴 버튼: 책 신청 (F-01) / 독후감 제출 (F-04) / 내 정보

### 사용자 흐름

1. 동호회 목록에서 무제 클릭 → 이 페이지
2. 책 신청 버튼 → 책 신청 페이지 (PAGE-04)
3. 독후감 버튼 → 독후감 제출 페이지 (PAGE-05)
4. 내 정보 버튼 → 내 정보 페이지 (PAGE-08)

---

## PAGE-04: 책 신청 페이지 (F-01)

- **목적**: 당월 도서 신청. URL 파싱으로 오입력 최소화
- **접근 권한**: 무제 동호회 회원만 접근

### 구성 요소

- 잔여 예산 표시 배너 (예: "이번 달 잔여 예산: 17,000원")
- 알라딘 URL 입력 필드 + 파싱 버튼
- 자동 입력 도서 정보 확인 영역 (제목, 저자, 가격, ISBN)
- 카테고리 선택 드롭다운
- 제출 버튼
- 이번 달 내 신청 목록 (하단)

### 사용자 흐름

1. 알라딘 URL 붙여넣기 → 파싱 버튼 클릭
2. 도서 정보 자동 입력 → 확인/수정
3. 카테고리 선택 → 제출
4. 예산 초과 시 오류 팝업 노출 ("잔여 예산 17,000원. 신청 금액 18,000원으로 초과입니다")
5. 신청 마감 후 신청 폼 비활성화 + 안내 문구 노출

### 예외 처리

- 알라딘 외 URL 입력 → "알라딘 URL만 지원합니다" 인라인 오류
- 이미 신청한 ISBN → "이미 이번 달 신청한 도서입니다" 오류

## PAGE-05: 독후감 제출 페이지 (F-04)

- **목적**: 당월 독후감 작성 및 제출
- **접근 권한**: 무제 동호회 회원만 접근

### 구성 요소

- 연결 도서 정보 표시 (제목, 저자)
- 독후감 제목 입력 필드
- 본문 텍스트 에디터
- 별점 선택 (1~5)
- 마감일 D-day 표시 배너 (예: "마감까지 3일 남았습니다")
- 제출 버튼

### 사용자 흐름

1. 이번 달 신청 도서 자동 연결
2. 제목 / 본문 / 별점 입력
3. 제출 → 성공 시 완료 메시지 + 무제 대시보드로 복귀
4. 마감일 이후 접근 시 폼 비활성화 + "독후감 제출 기간이 종료되었습니다" 안내

### 예외 처리

- 당월 책 신청 없는 회원 → "이번 달 신청한 도서가 없습니다" + 책 신청 페이지 링크
- 이미 제출한 경우 → 기제출 독후감 조회 모드로 전환 + 수정 버튼 노출 (마감 전)

---

## PAGE-06: 벌점 이력 페이지 (F-07)

- **목적**: 본인 벌점 이력 확인 및 이의 제기
- **접근 권한**: 로그인 필수. 본인 데이터만 조회 가능

### 구성 요소

- 누적 벌점 요약 (이번 달 / 전체)
- 벌점 이력 목록 (일시, 타입, 사유, 상태)
- 이의 제기 버튼 (처리 상태 표시: 처리 중 / 수락 / 기각)

### 사용자 흐름

1. 벌점 이력 목록 확인
2. 특정 벌점 항목 클릭 → 상세 사유 확인
3. 이의 제기 버튼 클릭 → 사유 입력 모달 → 제출
4. 이의 제기 처리 결과 슬랙 DM 수신 → 페이지 재접속 시 결과 확인

### 예외 처리

- 이미 이의 제기 중인 항목 재신청 → "처리 중인 이의 제기가 있습니다" 안내

## PAGE-07: 관리자 대시보드

- **목적**: 운영진 전용 관리 화면. 월간 운영 현황 한눈에 파악
- **접근 권한**: 관리자만 접근. 일반 회원 접근 시 권한 없음 안내

### 구성 요소

- 월 선택 필터
- 현황 요약 카드 3개: 책 신청 현황 / 독후감 제출율 / 미수령 인원
- 미제출자 목록 (이름, 항목별 현황)
- 벌점 부여 바로가기 버튼
- 슬랙 공지 발송 버튼 (타겟 메시지 수동 발송)

### 사용자 흐름

1. 관리자 로그인 → 상단 메뉴에서 "관리자 대시보드" 진입
2. 월 선택 → 해당 월 현황 즉시 갱신
3. 미제출자 항목 클릭 → 해당 회원 상세 이동
4. 벌점 부여 버튼 → 벌점 부여 모달 (회원 선택 / 타입 선택 / 사유 입력)
5. 슬랙 공지 버튼 → 메시지 입력 모달 → 채널/DM 선택 → 발송

### 예외 처리

- 해당 월 운영 일정 미등록 상태 → 경고 배너 노출 ("이번 달 일정이 등록되지 않았습니다")

## PAGE-08: 내 정보 페이지

- **목적**: 프로필 확인/수정, 포인트·벌점 요약, 신청 이력 확인
- **접근 권한**: 로그인 필수. 본인 데이터만 조회

### 구성 요소

- 프로필 섹션: 이름(수정 불가) / 닉네임 / 부서 / 슬랙 사용자 ID 입력
- 이번 달 요약: 잔여 예산 / 누적 포인트 / 현재 벌점
  - 포인트·벌점은 동호회별 탭으로 구분 표시. 복수 동호회 가입 시 탭 전환으로 각 동호회 현황 확인
- 탭 메뉴: 책 신청 이력 / 독후감 이력 / 벌점 이력 (PAGE-06 연결)

### 사용자 흐름

1. 프로필 수정 → 저장 버튼 클릭 → 즉시 반영
2. 탭 전환으로 이력 확인
3. 벌점 이력 탭 → PAGE-06 기능과 동일

### 예외 처리

- 슬랙 사용자 ID 미등록 상태 → 경고 문구 ("슬랙 ID 미등록 시 DM 알림을 받을 수 없습니다")
